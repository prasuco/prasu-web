import { Target, Users, BookOpen, Clock, Heart } from 'lucide-react';

Building an open source community is more than just writing code and pushing it to GitHub. Over the past year of running Prasuco, I've learned valuable lessons about what it takes to create a thriving open source initiative.

## Start with a Clear Vision

<div className="flex items-start gap-3 bg-blue-50 p-4 rounded-lg my-6">
  <Target className="h-6 w-6 text-blue-600 mt-1 flex-shrink-0" />
  <div>
    <p className="text-blue-800">
      The most successful open source projects have a clear, compelling vision. For Prasuco, our vision was simple: create better end-user software that is free, open-source, and easy to use. This clarity helps attract contributors who share your values.
    </p>
  </div>
</div>

## Create Welcoming Spaces

<div className="flex items-start gap-3 bg-green-50 p-4 rounded-lg my-6">
  <Users className="h-6 w-6 text-green-600 mt-1 flex-shrink-0" />
  <div>
    <p className="text-green-800">
      Technical excellence alone isn't enough. You need to create spaces where people feel welcome to contribute, regardless of their skill level. This means:
    </p>
    <ul className="mt-3 space-y-2 text-green-700">
      <li>• Clear contribution guidelines</li>
      <li>• Responsive maintainers</li>
      <li>• Code reviews that teach rather than criticize</li>
      <li>• Celebrating all contributions, not just code</li>
    </ul>
  </div>
</div>

## Documentation is Your Best Friend

<div className="flex items-start gap-3 bg-purple-50 p-4 rounded-lg my-6">
  <BookOpen className="h-6 w-6 text-purple-600 mt-1 flex-shrink-0" />
  <div>
    <p className="text-purple-800">
      Good documentation is often the difference between a project that thrives and one that dies. We invest time in:
    </p>
    <ul className="mt-3 space-y-2 text-purple-700">
      <li>• Clear README files</li>
      <li>• Comprehensive setup guides</li>
      <li>• API documentation</li>
      <li>• Contribution guides</li>
    </ul>
  </div>
</div>

## Consistency Builds Trust

<div className="flex items-start gap-3 bg-orange-50 p-4 rounded-lg my-6">
  <Clock className="h-6 w-6 text-orange-600 mt-1 flex-shrink-0" />
  <div>
    <p className="text-orange-800">
      Regular releases, consistent communication, and predictable maintenance schedules build trust with your community. People need to know that your project is actively maintained.
    </p>
  </div>
</div>
